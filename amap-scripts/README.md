# AMAP Scripts (backup copy)

This is a backup/reference copy of the actual AMAP server-management scripts running on the Rust server, kept here so admins have a copy to restore from if the live ones on the server ever need to be replaced. It mirrors the real folder layout (`AMAP.sh`, `config.sh`, `Files/Scripts/`, `Files/Config/`) so it can be dropped back onto the server as a starting point if needed.

This is **not** the live, actively-maintained version of AMAP - that lives in its own separate repo. This copy can drift out of date; check there first if you need the latest version of a specific script.

## Before using any of this for real

Two files had real secrets in them on the live server and have been replaced with `CHANGE_ME` placeholders here, since this repo is public:

- **`config.sh`**: `DISCORDURL` - your Discord webhook URL for wipe/alert notifications.
- **`Files/Config/templateconfig.cfg`**: `rconpassword` and `discordwebhook` - your real RCON password and a second webhook used directly in the generated server config.

`Fullwipe.sh` and `Mapwipe.sh` were also changed slightly: they used to have their own hardcoded Discord webhook (a third, different one) instead of using `config.sh`'s `DISCORDURL` - they now reference `$DISCORDURL` instead, so there's just one webhook to set instead of three.

Fill in those placeholders with your real values before relying on these scripts, and make sure `USERNAME` and `HOSTNAME` near the top of `config.sh` match your actual server setup.

## What's in here

- **`AMAP.sh` / `AMAPNC.sh`** - the main menu entry points.
- **`config.sh`** - shared settings and path variables every other script sources.
- **`Files/Scripts/`** - the actual action scripts (backups, wipes, updater, server start/stop helpers, etc.) - the same ones the NOR Dashboard's AMAP tab runs over RCON via `plugins/AmapBridge.cs`.
- **`Files/Config/templateconfig.cfg`** / **`templatesmartbot.json`** - templates `wipeconfigure.sh` fills in for each new wipe.

Not included: the live, wipe-specific generated config files (`common.cfg`, `config.cfg`, `rustserver.cfg`) - those hold a live RCON password and aren't reusable "scripts," just per-wipe state that gets regenerated by `wipeconfigure.sh` anyway.
